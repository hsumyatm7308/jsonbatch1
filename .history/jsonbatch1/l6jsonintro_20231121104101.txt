=>JSON (Javascript Object Notation )

=JSON 
Javascript Object Notation 
Base on Text Format 
Light-weighted 
Does not support comments and namespace 
NoSQL database like MongoDB use JSON formats 

=>JSON 
Need to enclose name of key within double quotes (should not use single quotes)
But can use single quotes to enclose key names 


=> JSON vs XML 
JSON is more condise /lightweight / easy to unserstand  
XML is more text / longer and look more complex 


=JSON Syntax Rules 
JSON data to written inside curly braces 
Data is represented as key value pairs 
Keys should be enclose in double quotes 
Key and value should be separated by column (:)
Comma (,) is used to separate data
